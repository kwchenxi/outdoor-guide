# 兼容性构建说明

## 问题

默认情况下，Vite 使用 ES 模块格式构建项目，这会在生成的 HTML 文件中包含 `<script type="module">` 标签。这种格式在一些旧版浏览器中不被支持，可能导致应用无法正常运行。

## 解决方案

我们提供了一个兼容性构建脚本，它会创建一个兼容性更好的构建版本，具有以下特点：

1. **使用 IIFE 格式**：立即执行函数表达式格式，兼容性更好
2. **移除 type="module" 属性**：使用普通的 `<script>` 标签
3. **目标浏览器为 ES2015+**：兼容大多数现代浏览器
4. **代码不分割**：减少请求数量，提高加载速度

## 使用方法

运行以下命令进行兼容性构建：

```bash
npm run build:compat
```

或者直接执行脚本：

```bash
./build-compat.sh
```

## 技术细节

兼容性构建主要做了以下修改：

1. **HTML 模板**：使用 `index-compat.html` 作为模板，移除了 type="module" 属性
2. **Vite 配置**：
   - 设置 `target: 'es2015'`，支持更多浏览器
   - 设置 `format: 'iife'`，使用立即执行函数表达式格式
   - 禁用代码分割，减少请求数量
3. **后处理**：构建完成后自动移除剩余的 type="module" 属性

## 浏览器兼容性

兼容性构建后的代码支持以下浏览器：

- Chrome 51+
- Firefox 54+
- Safari 10+
- Edge 15+
- IE 11（需要 polyfills）

## 注意事项

1. 兼容性构建会生成较大的单个 JavaScript 文件，因为代码没有分割
2. 如果您的应用非常复杂，可能需要手动添加 polyfills
3. 对于现代浏览器环境，建议使用标准构建（`npm run build`）以获得更好的性能
